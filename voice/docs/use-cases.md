# Use Cases

## 1. 혼자 앱을 만드는 솔로 크리에이터

> "앱은 만들었는데, 어디서부터 개선해야 할지 모르겠어요."

**상황:** 바이브코딩으로 웹앱을 하나 만들었다. 동작은 하는데 코드 품질이 어떤지, 뭘 고쳐야 하는지 감이 안 온다.

**Comad Voice 사용:**

```
검토해봐
```

**결과:**

```
카드 1: API 에러 처리 — 난이도 ⭐ / 효과 🔥🔥🔥
  현재 try-catch 없이 API 호출. 서버 다운 시 앱이 멈춤.

카드 2: 중복 코드 정리 — 난이도 ⭐⭐ / 효과 🔥🔥
  같은 패턴이 5곳에서 반복됨. 유틸리티로 추출 가능.

카드 3: 로딩 UX — 난이도 ⭐ / 효과 🔥🔥
  데이터 로딩 중 빈 화면. 스켈레톤 UI 추가 권장.
```

"1번" 선택 → AI가 자동으로 에러 처리 추가, 테스트, 검증까지 완료.

**이전과 차이:** "에러 처리 추가해줘"라고 정확히 알아야 시킬 수 있었음. 이제는 "검토해봐" 한마디로 AI가 문제를 찾아줌.

## 2. 토큰을 낭비하는 구독자

> "Claude Max를 결제했는데 매달 토큰의 10%도 못 쓰고 있어요."

**상황:** 월 $100 이상 결제하는데, "이거 고쳐줘" "저거 고쳐줘" 수준의 단발성 요청만 한다. AI의 진짜 능력을 10%도 활용 못 하고 있다.

**Comad Voice 사용:**

```
풀사이클로 이 프로젝트 전체 품질을 올려줘
```

**결과:** 6단계 파이프라인이 자동 실행되면서:
- RESEARCH: 코드 분석 + 관련 기술 리서치
- DECOMPOSE: 5개 서브태스크 도출 (3개 독립, 2개 의존)
- EXPERIMENT: 독립 3개는 Codex에 병렬 위임, 의존 2개는 Claude가 순차 실행
- INTEGRATE: 최적 결과 병합
- POLISH: QA + 문서화
- DELIVER: PR 생성

**Opus 토큰을 마음껏 쓰면서** 프로젝트 전체를 체계적으로 개선.

**이전과 차이:** 뭘 시켜야 할지 몰라서 토큰이 남았음. 이제는 "풀사이클" 한마디로 토큰을 유의미하게 소진.

## 3. 멀티 AI 구독자

> "Claude, ChatGPT, Gemini 다 있는데 각각 따로 쓰고 있어요."

**상황:** AI 서비스를 3개나 구독하는데, 각각 독립적으로 사용. 서로의 강점을 조합해서 쓰는 방법을 모름.

**Comad Voice 사용:**

대주제를 던지면 자동으로 멀티-AI 오케스트레이션:

- **Claude (Opus)**: 전체 설계 + 의사결정 + 핵심 구현
- **Codex**: 독립적인 모듈들을 병렬로 구현
- **Gemini**: 대규모 문서/논문 분석, 리서치

사용자가 "이건 Claude한테, 저건 Codex한테" 판단할 필요 없음.
5가지 체크리스트로 AI가 자동 분류.

**이전과 차이:** 3개 AI를 따로 썼음. 이제는 하나의 명령으로 3개가 협업.
