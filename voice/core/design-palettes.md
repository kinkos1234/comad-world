# Comad Voice Design Palettes

T6 디자인 파이프라인에서 사용하는 컬러 팔레트 레퍼런스.
프로젝트 컨셉에 맞는 팔레트를 자동 선택하여 디자인 시안에 적용한다.

출처: flatuicolors.com + Material UI

---

| 프로젝트 컨셉 | 추천 팔레트 | 이유 |

## 타이포그래피 계층 가이드

10개 디자인 시스템/프로덕트 분석 기반 (Tailwind, shadcn/ui, Ant Design, Chakra UI, Linear, Stripe, Vercel, Notion, Airbnb, GitHub Primer).

### 폰트 크기 스케일 (권장 기본값)

모든 주요 디자인 시스템이 수렴하는 공통 스케일:

| 레벨 | 크기 | 행간 (line-height) | 용도 | 무게 |
|------|------|-------------------|------|------|
| **Display** | 48-72px (3-4.5rem) | 1.0-1.1 | 히어로 헤드라인, 랜딩 페이지 | 700-800 |
| **H1** | 36-48px (2.25-3rem) | 1.1-1.2 | 페이지 제목 | 700 |
| **H2** | 28-36px (1.75-2.25rem) | 1.2-1.3 | 섹션 제목 | 600-700 |
| **H3** | 22-28px (1.375-1.75rem) | 1.3-1.4 | 서브섹션 제목 | 600 |
| **H4** | 18-22px (1.125-1.375rem) | 1.4-1.5 | 카드 제목, 사이드바 제목 | 600 |
| **Body L** | 18px (1.125rem) | 1.6-1.75 | 리드 문단, 강조 본문 | 400 |
| **Body** | 16px (1rem) | 1.5-1.6 | 기본 본문 | 400 |
| **Body S** | 14px (0.875rem) | 1.5-1.6 | 보조 본문, 설명 텍스트 | 400 |
| **Caption** | 13px (0.8125rem) | 1.4-1.5 | 메타데이터, 타임스탬프 | 400-500 |
| **Label** | 12px (0.75rem) | 1.4 | 뱃지, 태그, 작은 라벨 | 500 |
| **Micro** | 11px (0.6875rem) | 1.3 | 법적 고지, 최소 텍스트 | 400 |

### 폰트 무게 (Font Weight) 체계

| 무게 | 값 | 용도 |
|------|-----|------|
| **Regular** | 400 | 본문, 설명 텍스트, 기본값 |
| **Medium** | 500 | 라벨, 네비게이션, 약한 강조 |
| **Semibold** | 600 | 서브 헤딩, 버튼, 중간 강조 |
| **Bold** | 700 | 메인 헤딩, CTA, 강한 강조 |

> 대부분의 프로덕트는 3-4개 무게만 사용. 5개 이상은 계층이 복잡해짐.
> Stripe/Linear: 300-400-500-600-700 (5단계)
> Vercel/Notion: 400-500-600-700 (4단계)

### 행간 (Line Height) 규칙

```
헤딩 (36px+):  1.0 - 1.2  (타이트하게)
서브헤딩:       1.2 - 1.4  (중간)
본문:           1.5 - 1.6  (읽기 편하게)
리스트/UI:      1.4 - 1.5  (컴팩트)
```

> 폰트 크기가 클수록 행간 비율은 줄어든다 (Tailwind, Ant Design 공통 패턴).

## 모서리 반경 (Border Radius) 체계

| 토큰 | 값 | 용도 |
|------|-----|------|
| **none** | 0 | 직각 요소, 구분선 |
| **sm** | 4px (0.25rem) | 뱃지, 태그, 작은 요소 |
| **md** | 6px (0.375rem) | 입력 필드, 작은 버튼 |
| **DEFAULT** | 8px (0.5rem) | 버튼, 카드 |
| **lg** | 12px (0.75rem) | 카드 (대), 모달 |
| **xl** | 16px (1rem) | 대형 카드, 이미지 |
| **2xl** | 24px (1.5rem) | 알약형 버튼, 칩 |

## 프로젝트 유형별 권장 조합

### SaaS / 대시보드 (Linear, Stripe 스타일)

```
폰트: Inter (본문) + Inter (제목)
헤딩: H1=36px/700, H2=28px/600, H3=22px/600
본문: 14px/400 (대시보드는 정보 밀도 높이므로 14px 기본)
간격: 컴팩트 (8-12-16px)
반경: 8px
텍스트 4단계: primary/secondary/tertiary/disabled
배경: white + gray-50 교차
```

### 소비자 앱 / 랜딩 (Notion, Airbnb 스타일)

```
폰트: Inter 또는 시스템 폰트 (본문) + 특색 폰트 (제목)
헤딩: Display=48-72px/800, H1=36px/700, H2=28px/600
본문: 16-18px/400 (읽기 편한 크기)
간격: 여유 (16-24-48px)
반경: 12px
텍스트 3단계: primary/secondary/muted
배경: white + 브랜드 컬러 액센트 섹션
```

### 개발자 도구 / 기술 (Vercel, GitHub 스타일)

```
폰트: Geist/Inter (본문) + Geist Mono (코드)
헤딩: H1=36px/700, H2=24px/600, H3=20px/600
본문: 15-16px/400
간격: 중간 (12-16-32px)
반경: 8px
텍스트 3단계: primary/secondary/tertiary
배경: 다크 모드 기본 + 라이트 모드 지원
코드 블록: 모노폰트, 14px, 배경 gray-900
```

## 커스텀 팔레트 생성 규칙

위 레퍼런스에 맞는 팔레트가 없을 때:

1. **60-30-10 법칙**: 주색 60%, 보조색 30%, 강조색 10%
2. **대비 비율**: 텍스트-배경 간 최소 4.5:1 (WCAG AA), 대형 텍스트 3:1
3. **시맨틱 컬러 필수**: success(녹색), warning(노란색), error(빨간색), info(파란색)
4. **중립색 11단계**: Tailwind 방식 50-950 (밝은 배경부터 어두운 텍스트까지)
5. **접근성 검증**: 색각이상자를 위해 색상만으로 정보 구분하지 않기
6. **라이트/다크 모드 쌍**: 모든 시맨틱 토큰에 라이트/다크 값 정의
