### T3. 병렬 자동 감지

분해(DECOMPOSE) 단계에서 5-Point Checklist로 의존성 자동 분석.
사용자가 의존성을 판단할 필요 없음. Claude가 자동 체크:

| # | 질문 | 독립 | 의존 |
|---|------|------|------|
| 1 | **파일 겹침**: 다른 태스크와 같은 파일을 수정? | 없음 | 있음 |
| 2 | **인터페이스 의존**: 새로 만들어질 함수를 호출? | 아니오 | 예 |
| 3 | **데이터 흐름**: 다른 태스크 출력이 입력? | 아니오 | 예 |
| 4 | **실행 순서**: 반드시 다른 태스크 이후? | 아니오 | 예 |
| 5 | **상태 공유**: 공유 상태를 변경? | 아니오 | 예 |

### 분류 결과 및 실행 전략

- **독립 태스크 2개 이상** → Agent tool로 병렬 에이전트 생성하여 동시 실행
- **의존 태스크** → Claude가 의존성 순서에 따라 순차 실행
- **독립이지만 맥락 필요** → Claude가 직접 처리

### 병렬 실행 방법

Claude Code의 Agent tool을 사용하여 병렬 처리:

```
# 독립 태스크를 동시에 실행하는 예시
Agent(subagent_type="general-purpose", model="sonnet",
      prompt="실행자 역할. [S1] 태스크 내용...")
Agent(subagent_type="general-purpose", model="sonnet",
      prompt="실행자 역할. [S3] 태스크 내용...")
# → 두 에이전트가 동시에 실행됨
```

### Codex CLI 활용 (선택)

Codex CLI가 설치되어 있으면 독립 태스크를 Codex에 위임 가능:

| 에이전트에 넘기기 좋은 작업 | Claude가 직접 하는 게 나은 작업 |
|--------------------------|-------------------------------|
| 새 유틸리티 함수/모듈 작성 | 기존 코드 깊은 리팩토링 |
| 독립된 UI 컴포넌트 구현 | 여러 파일에 걸친 아키텍처 변경 |
| 테스트 코드 작성 | 복잡한 비즈니스 로직 수정 |
| 문서/README 생성 | 성능 최적화 (프로파일링 필요) |
| 데이터 변환 스크립트 | 디버깅/에러 추적 |
| 설정 파일/스키마 작성 | 보안 관련 코드 |

### 내레이션

분류 완료 후 사용자에게 실행 계획을 보고:

```
[분해 완료] 서브태스크 5개로 나눴습니다.

독립 태스크 3개 → 병렬 실행 (에이전트 3개 동시 생성)
의존 태스크 2개 → S1 완료 후 순차 실행

이대로 진행할까요?
```
