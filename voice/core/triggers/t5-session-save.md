### T5. 세션 저장 & 인수인계

**감지 키워드:** "저장해줘", "여기까지", "다음에 이어서", "세션 끝", "오늘은 여기까지", "save session", "인수인계"

**실행 절차:**

1. **작업 요약 자동 생성**

   현재 세션에서 수행한 작업을 자동으로 추적:
   - 수정한 파일 목록 (git diff --stat)
   - 실행한 실험과 결과 (keep/discard)
   - 내린 결정과 이유
   - 발견한 문제점

2. **세션 파일 저장**

   `.comad/sessions/YYYY-MM-DD-HHmm.md` 형식으로 저장:

   ```markdown
   ## 세션 저장 — 2024-03-25 14:30

   ### 완료한 작업
   - [작업 1 요약]
   - [작업 2 요약]

   ### 미완료 / 다음에 할 것
   - [ ] [남은 작업 1]
   - [ ] [남은 작업 2]

   ### 핵심 결정 사항
   - [결정 1]: [이유]
   - [결정 2]: [이유]

   ### 실험 결과 요약 (있을 경우)
   | 실험 | 가설 | 결과 | keep/discard |
   |------|------|------|-------------|
   | 1 | ... | ... | keep |
   | 2 | ... | ... | discard |

   ### 주의할 점
   - [발견된 이슈나 주의사항]

   ### 수정된 파일
   - [git diff --stat 결과]
   ```

3. **메모리 업데이트**

   `memory/MEMORY.md`에 현재 상태 자동 반영:
   - Current State 섹션 업데이트
   - 중요한 결정 사항 기록
   - Known Issues에 발견된 문제 추가

4. **`.comad/state.json` 업데이트**

   ```json
   {
     "lastSession": "2024-03-25T14:30:00Z",
     "lastSessionFile": ".comad/sessions/2024-03-25-1430.md",
     "pendingTasks": ["남은 작업 1", "남은 작업 2"],
     "sessions": 5,
     "experiments": 12
   }
   ```

5. **인수인계 안내 출력**

   ```
   세션이 저장되었습니다.

   저장 위치: .comad/sessions/2024-03-25-1430.md
   완료: 3개 작업
   미완료: 2개 작업

   다음 세션에서 "이어서 해줘"라고 말하면
   여기서부터 바로 시작합니다.
   ```

### "이어서 해줘" 감지 (T5 연계)

**감지 키워드:** "이어서 해줘", "어디까지 했어?", "지난번 이어서", "continue", "resume"

**실행:**
1. `.comad/sessions/` 에서 가장 최근 세션 파일 로드
2. 미완료 작업 목록 제시
3. "어디서부터 이어갈까요?" 확인
4. 선택된 작업부터 자동 재개
