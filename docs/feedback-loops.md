# Feedback Loops — Beyond the Linear Pipeline

Wilson 관점에서 보면, 건강한 시스템은 단일 포식 사슬보다 상호공생하는 생태계에 가깝다.

## 현재 파이프라인: ear → brain → eye (단방향 food chain)

Comad World v0.2.0의 기본 흐름은 `ear`가 수집하고, `brain`이 구조화하고,
`eye`가 읽는 선형 파이프라인에 가깝다. 데이터는 앞으로만 흐르고, 뒤쪽
시스템이 앞쪽 시스템의 선택 기준을 학습시키는 역방향 상호작용은 거의 없다.

## 문제: 생태계가 아닌 사슬, 종 간 상호공생 부족

지금 구조는 각 패키지가 자기 역할을 잘 수행해도 전체 시스템이 스스로
생태화되지는 않는다.

- `ear`는 수집량은 늘릴 수 있지만 무엇을 더 먼저 봐야 하는지 뒤쪽 신호를 거의 받지 못한다.
- `brain`은 그래프를 더 풍부하게 만들 수 있지만 새로운 feed 발굴을 직접 유도하지 못한다.
- `eye`는 어떤 렌즈가 실제로 유용했는지 알 수 있어도 그 결과가 다음 크롤 우선순위로 연결되지 않는다.
- `sleep`과 `photo` 같은 느린 배치성 모듈도 독립적으로 돌 뿐, 다른 워크플로를 예열하거나 호출하는 역할이 약하다.

결과적으로 시스템은 먹이사슬처럼 이어져 있지만, 숲처럼 서로를 살리는 구조는 아니다.

## 제안 reverse edges

- `eye → ear`: 예측 정확도 높은 lens가 자주 참조하는 소스를 `ear` 크롤 우선순위로 역주입한다.
  정확도가 높은 렌즈가 반복적으로 인용하는 출처는 이미 downstream에서 효용이 검증된 셈이므로, upstream 수집 가중치로 환원할 가치가 높다.
- `brain → ear`: graph hub 노드 근방 토픽을 신규 RSS feed 후보로 추천한다.
  최근 연결이 급증한 허브 근처의 토픽은 아직 직접 소스가 부족한 영역일 가능성이 높아, `ear`의 소스 발굴 표면으로 보내기 적합하다.
- `sleep → brain`: 세션 패턴이 graph 쿼리 캐시를 warming 한다.
  사용자가 밤사이에 자주 열었던 경로와 반복 질의를 이용하면, 다음 세션 전에 GraphRAG 관련 캐시를 미리 데워 응답 초반 지연을 줄일 수 있다.
- `photo → voice`: 이미지 처리 완료 이벤트가 voice workflow를 트리거한다.
  시각 파이프라인이 끝난 뒤 음성 요약이나 설명 워크플로를 자동 호출하면, 동일한 산출물을 다른 modality로 재사용할 수 있다.

## 구현 우선순위 표 (난이도 × 효과)

| Reverse Edge | 기대 효과 | 난이도 | 우선순위 |
| --- | --- | --- | --- |
| `eye → ear` | 실제로 잘 맞는 소스를 더 자주 수집하게 되어 precision 개선 효과가 즉각적 | 중 | 1 |
| `brain → ear` | 그래프에서 떠오르는 허브 주변을 신규 feed 발굴 루프로 연결 | 중 | 2 |
| `sleep → brain` | 반복 질의 latency 완화, 아침 첫 응답 품질 안정화 | 중하 | 3 |
| `photo → voice` | 멀티모달 재사용성 확대, 자동 후속 워크플로 생성 | 하 | 4 |

우선순위 판단 기준은 다음 두 축이다.

- 효과: 수집 품질, 질의 응답성, 워크플로 연결성에 주는 시스템 전체 이득
- 난이도: 새 상태 저장소, 이벤트 버스, 편향 완화 로직이 얼마나 필요한가

## 위험: 순환 피드백으로 bias 증폭 가능성 + 완화책

역방향 edge가 많아질수록 시스템은 더 똑똑해질 수 있지만, 동시에 자기 자신이
좋아하던 것만 더 자주 먹는 강화 루프로 빠질 수 있다.

| 위험 | 증상 | 완화책 |
| --- | --- | --- |
| 인기 소스 편향 고착 | 이미 많이 인용된 소스만 계속 상위 우선순위를 차지 | exploration budget을 별도 유지하고, 일정 비율은 미검증 신규 소스에 강제 할당 |
| 허브 노드 과증폭 | 연결 수가 많은 토픽만 계속 feed 추천에 재사용 | 최근 증가율과 novelty score를 같이 보고, 절대 degree만으로 추천하지 않기 |
| 렌즈 자기강화 | 특정 lens의 오탐이 수집 정책에 재주입 | lens 정확도 하한선, holdout 평가셋, 수동 샘플링 리뷰로 역주입 조건 제한 |
| 캐시 오염 | `sleep → brain` 예열이 오래된 패턴을 굳힘 | TTL 기반 캐시와 세션 신선도 가중치로 오래된 패턴 자동 감쇠 |
| 이벤트 폭주 | `photo → voice`가 연쇄 트리거를 일으켜 큐를 점유 | idempotency key, debounce, workflow depth limit로 재귀성 차단 |

핵심은 reverse edge를 “자동 강화”가 아니라 “가설 생성 + 제한적 역주입”으로
다루는 것이다. 생태계는 순환이 있어야 강해지지만, 모든 순환이 건강한 것은 아니다.
