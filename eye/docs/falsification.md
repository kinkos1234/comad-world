# 반증 프로토콜: Lens 예측의 인식론적 위생

## 목적

`eye`의 렌즈 분석은 설명력이 높을수록 오히려 자기확신 오류를 만들기 쉽다. 따라서 Popper식 반증 원칙을 운영 규칙으로 명문화한다. 핵심은 "잘 맞은 해석을 칭찬하기보다 틀린 예측을 빠르게 벌점화하는 것"이다. 렌즈는 통찰 도구이지 교리가 아니며, 검증 가능한 미래 지시적 진술만 평가 대상이 된다.

## 1. weight 감쇠 규칙

각 렌즈 또는 렌즈-공간 조합은 `weight`를 가진다. 예측이 사후 검증에서 틀린 것으로 판정되면 다음 규칙으로 가중치를 감쇠한다.

```text
w_new = w_old * decay^n
```

- 제안 기본값: `decay = 0.9`
- `n`: 누적 반증 횟수. 동일 렌즈가 연속으로 틀렸는지 여부와 무관하게 검증 가능한 실패 건수를 누적한다.
- 예시: `w_old=0.80`, `n=1`이면 `0.72`, `n=3`이면 `0.80 * 0.9^3 = 0.5832`
- 운영 권고: `weight < 0.5`가 되면 기본 자동선택 대상에서 제외하고, `weight < 0.3`이면 수동 요청 시에만 활성화한다.
- 성공 예측 보상은 보수적으로 적용한다. 반증 페널티보다 큰 폭의 보상은 금지하며, 회복은 주간 리뷰에서만 승인한다. 이유는 과적합된 최근 적중이 장기 신뢰도를 왜곡할 수 있기 때문이다.

이 규칙은 "한 번의 실패"보다 "반복적으로 틀리는 패턴"을 더 크게 벌한다. 즉, 반증 가능성과 갱신 가능성이 렌즈 품질의 핵심 지표다.

## 2. Prediction log 스키마

반증 가능한 예측만 아래 JSON 스키마로 기록한다.

```json
{
  "ts": "2026-04-14T09:00:00Z",
  "lens_id": "taleb",
  "prediction": "향후 7일 내 단일 브릿지 노드 장애가 전체 연결성 하락을 유발한다",
  "outcome": "false",
  "delta": -0.08,
  "weight_before": 0.8,
  "weight_after": 0.72
}
```

필드 정의:
- `ts`: 판정 시각. 예측 생성 시각이 아니라 outcome을 확정한 시각을 기록한다.
- `lens_id`: 렌즈 ID 또는 필요 시 `lens_id:space` 형식의 세부 키.
- `prediction`: 기간, 조건, 관찰지표가 포함된 문장형 예측.
- `outcome`: `true`, `false`, `mixed` 중 하나를 권고한다. 다만 `mixed`는 weight 자동조정 없이 리뷰 큐로 보낸다.
- `delta`: 관찰치와 기준치의 차이. 예를 들어 연결성, volatility, claim confidence 등 실제 측정값의 변화량.
- `weight_before`, `weight_after`: 갱신 전후 가중치.

## 3. 로그 제외 규칙

다음 조건 중 하나라도 해당하면 기록하지 않는다.
- `verifiable`하지 않은 예측. 예: "시스템이 더 지혜로워질 것이다"처럼 관찰 가능한 판정 기준이 없는 진술.
- 시간 경계가 없는 예측. 예: "언젠가 붕괴한다".
- outcome 판정자를 사후에 임의 변경할 수 있는 예측.
- 사실상 회고적 해석인 문장. 예측이 아니라 설명이면 로그 대상이 아니다.

정리하면, 반증 불가능한 예측은 로그에 기록 금지다. 로그의 목적은 렌즈의 수사력을 축적하는 것이 아니라, 실패 가능한 주장만 남겨 모델의 오만을 줄이는 데 있다.

## 4. 주간 리뷰 절차

매주 1회, 최근 7일 prediction log를 검토한다.

1. `outcome=false` 비율이 높은 렌즈를 우선 정렬한다.
2. 실패 예측을 읽고, 실패 원인을 세 갈래로 분리한다: 데이터 부족, 시간창 설정 오류, 프레임 자체의 오판.
3. 동일 실패 패턴이 3건 이상 반복되면 해당 렌즈의 시스템 프롬프트 또는 선택 예산을 조정한다.
4. `mixed` 판정은 인간 리뷰어가 `true` 또는 `false`로 재분류하거나 폐기한다.
5. 4주 누적 기준으로 지속 하락하는 렌즈는 기본 렌즈 집합에서 제외 후보로 올린다.
6. 리뷰 결과는 "유지, 감쇠, 보류, 은퇴" 네 상태로 남기고 다음 주 자동선택 정책에 반영한다.

주간 리뷰의 목적은 점수표를 만드는 것이 아니라, 잘못된 렌즈를 빨리 약화시키고 반증 가능한 렌즈만 살아남게 하는 것이다. Popper식 위생은 높은 정확도보다 빠른 오류 수정 속도를 요구한다.
