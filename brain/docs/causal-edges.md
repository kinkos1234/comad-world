# Causal Edges 운영 규약

## 목적

Comad World 그래프는 관계를 많이 모을수록 풍부해지지만, 관계의 종류를 구분하지 않으면 "같이 나왔다"와 "원인이다"가 같은 무게로 취급된다. Pearl의 인과 구분과 Korzybski의 "지도는 영토가 아니다" 원칙을 결합해, 그래프 엣지를 단계적으로 관리하는 기준을 정한다.

## 1. 엣지 타입 분류

모든 관계는 최소한 아래 세 단계 중 하나로 표기한다.

### associative

공출현 또는 서술적 인접성이다. 같은 문서, 같은 문단, 같은 토픽 안에 함께 등장했지만 방향성과 메커니즘은 없다. 예: 한 기사에서 `GPU`와 `전력비용`이 함께 자주 언급됨.

### correlational

통계적 상관이나 시계열 동행이 관찰되지만, 개입 가능성은 확인되지 않았다. 예: 특정 모델 출시 이후 오픈소스 파인튜닝 저장소 수가 함께 증가함. 방향, 숨은 변수, 역인과 가능성을 열어둬야 한다.

### causal

개입 또는 반사실 조건을 상정했을 때 결과 변화가 설명되는 관계다. 예: 정책 변경, API 가격 인상, 시스템 설정 변경처럼 원인을 조작했을 때 downstream 지표가 반복적으로 달라진 경우다.

실무적으로는 `assoc -> corr -> causal` 순으로 승격하며, 기본값은 항상 더 약한 쪽에서 시작한다. 증거가 부족한데 causal로 시작하는 것은 금지한다.

## 2. 엣지 메타필드 제안

현재 그래프 엣지에는 아래 메타필드를 추가하는 것을 권고한다.

```ts
edge_kind: 'assoc' | 'corr' | 'causal'
evidence_count: number
last_updated_ts: string
```

- `edge_kind`: 관계의 인식론적 등급. 시각화 색상, 검색 필터, 추론 규칙의 기본 입력이 된다.
- `evidence_count`: 해당 관계를 지지하는 독립 증거 수. 문서 수, 실험 수, 개입 기록 수를 합산하되 중복 소스는 1건으로 압축한다.
- `last_updated_ts`: 마지막으로 증거를 재검토한 시각. 오래된 인과는 유지가 아니라 재검증 대상이다.

필요하면 여기에 `confidence`, `source_diversity`, `intervention_evidence`를 확장할 수 있지만, 최소 위 세 필드는 모든 엣지에 공통으로 유지하는 편이 좋다.

## 3. Temporal decay 규칙

Korzybski 관점에서 그래프는 현실의 축약본일 뿐이며, 오래된 지도는 현실을 오도할 수 있다. 따라서 노드와 엣지의 나이에 따라 weight를 감쇠한다.

- 기본 규칙: 최근 갱신 시점에서 멀어질수록 edge weight를 단계적으로 낮춘다.
- 예시 함수: `effective_weight = base_weight * freshness_decay(age_days)`
- 권고 해석:
  - 30일 이내: 원가중치 유지
  - 31~90일: 약한 감쇠
  - 91~180일: 중간 감쇠
  - 180일 초과: 강한 감쇠 및 재검토 큐 편입

시각화에서도 동일 원칙을 적용한다. 오래된 노드와 엣지는 삭제보다 `fade-out`으로 표현해, 사용자가 "존재는 하지만 현재성은 낮다"는 점을 즉시 인식하게 한다. 오래된 causal edge가 선명하게 보이는 상태는 지도와 영토를 혼동하게 만드는 대표적 UI 오류다.

## 4. 인과 엣지 승격 기준

상관 관계를 causal로 올리기 위한 최소 기준은 다음과 같다.

1. 독립 개입 증거가 3건 이상 있을 것.
2. 세 증거가 동일 출처 재서술이 아니라 서로 독립적일 것.
3. 시간 순서가 일관될 것. 원인이 결과보다 앞서야 한다.
4. 대체 설명이 최소 1회 이상 검토될 것. 숨은 변수 가능성을 기록으로 남긴다.
5. 최근성 기준을 통과할 것. 너무 오래된 증거만으로 causal 승격 금지.

즉, 권고 승격 조건은 `intervention evidence >= 3`이다. 이 기준을 넘기기 전까지는 강한 상관이 있어도 `corr`로 남겨야 한다.

## 5. 운영 원칙

- 추론 엔진은 `causal` 엣지와 `corr` 엣지를 구분해 사용한다.
- 리포트에서는 `assoc`를 설명 배경으로, `corr`를 가설 후보로, `causal`을 개입 후보로 제시한다.
- 오래된 causal은 자동 폐기하지 말고 시각적으로 약화한 뒤 재검증을 요구한다.

이 규약의 목표는 그래프를 빈약하게 만드는 것이 아니다. 오히려 관계의 힘을 구분해, 풍부한 그래프가 과장된 그래프가 되지 않도록 막는 데 있다.
