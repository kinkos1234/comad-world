# Model Card: eye-lens

## Model Details

- 모델명: `eye-lens`
- 구현 위치: `eye/src/comad_eye/analysis/lenses.py`
- 유형: LLM 프롬프트 기반 렌즈 해석 시스템
- 기본 코어 렌즈: `sun_tzu`, `adam_smith`, `taleb`, `kahneman`, `meadows`
- 입력: `hierarchy`, `temporal`, `recursive`, `structural`, `causal` 5개 분석공간 결과
- 출력: 렌즈별 인사이트와 교차 렌즈 종합

이 시스템은 원시 그래프나 시뮬레이션 로그를 직접 분류하는 모델이 아니다. 먼저 계산적 분석공간이 구조화한 결과를 만든 뒤, 각 렌즈가 이를 특정 사상가의 질문틀로 재해석한다. 따라서 성격상 예측기라기보다 프레임워크 기반 해석기다.

## Intended Use

적합한 사용:
- 동일한 시뮬레이션 결과를 여러 지적 프레임으로 재독해할 때
- 전략, 시장, 리스크, 인지편향, 시스템 개입점처럼 서로 다른 해석 축을 병렬로 비교할 때
- 사람이 바로 읽기 어려운 다차원 분석 결과를 개념적 언어로 압축할 때

부적합한 사용:
- 렌즈 결과를 객관적 사실 판정이나 단일 정답처럼 사용할 때
- 렌즈의 수사적 설득력을 곧바로 예측 정확도로 해석할 때
- 개입 정책을 자동 실행하는 의사결정 엔진으로 사용할 때

## Factors

성능과 품질에 영향을 주는 요인:
- 5개 분석공간 결과의 품질. 기초 분석이 흔들리면 렌즈 해석도 함께 흔들린다.
- 선택된 렌즈 조합. 서로 보완적인 조합인지, 같은 편향을 반복하는지에 따라 결과가 달라진다.
- 프롬프트 설계. 사상가 프레임을 얼마나 좁고 검증 가능하게 정의했는지 중요하다.
- 그래프/RAG 보강 여부. 렌즈 원리 조회가 가능할 때 설명 일관성이 높아질 수 있다.
- 도메인 적합성. 지정학적 사건에는 손자 렌즈가 강할 수 있지만, 운영 병목 분석에는 메도우즈가 더 낫다.

## Metrics

현재 시스템의 핵심 평가지표는 전통적 정확도보다 운영 품질 지표에 가깝다.

- Lens Coverage: 5개 공간 중 각 렌즈가 유효 인사이트를 생성한 비율
- Cross-Lens Consistency: 서로 다른 렌즈가 동일 사건을 완전히 모순되게 해석하는 빈도
- Actionability: 사람이 읽고 후속 질문이나 개입 후보를 만들 수 있는 정도
- Falsification Readiness: 예측형 진술이 사후 검증 가능한 형태로 작성된 비율
- Reviewer Agreement: 인간 리뷰어가 "유용함"에 합의하는 비율

## Training Data

전통적 학습 데이터셋은 없다. 시스템은 사상가별 시스템 프롬프트와 공간별 프롬프트, 그리고 선택적으로 렌즈 원리 지식 그래프를 사용한다. 즉, 파라미터 학습보다는 프롬프트 설계와 맥락 주입이 성능을 결정한다. 이 때문에 데이터 편향보다 프레임 편향이 더 직접적이다.

## Evaluation Data

평가 입력은 분석공간이 생산한 구조화 결과와 테스트 더블(mock) 응답이다. 일부 테스트는 렌즈 자동선택, 공간별 적용, 교차 종합 흐름을 검증하지만, 인간 전문가 라벨이 붙은 gold evaluation set은 아직 없다. 따라서 품질 평가는 기능 검증 수준에 가깝고 해석의 타당성을 충분히 보증하지 않는다.

## Ethical Considerations

렌즈 시스템은 설명이 아름답게 나올수록 과신 위험이 커진다. 특히 손자, 탈레브, 카너먼 같은 강한 프레임은 사용자가 사건을 그 관점으로만 보게 만들 수 있다. 또한 동일 사건을 권력, 시장, 리스크, 편향의 언어로 번역하는 과정에서 실제 행위자와 맥락의 복잡성이 축소될 수 있다. 따라서 렌즈 출력은 보조 해석이어야 하며, 반증 기록과 인간 리뷰가 동반되어야 한다.

## Caveats

- 5 lens 시스템은 다양성을 늘리지만, 프레임 선택 자체가 편향을 내장한다.
- 렌즈 간 독립성이 완전하지 않아 유사한 결론이 반복될 수 있다.
- LLM 기반 해석이라 문장 유창성이 사실성보다 높게 보일 수 있다.
- 교차 렌즈 종합은 일관성을 주지만, 소수 관점을 평균화해 날카로운 반례를 지울 수 있다.
- 적용 범위는 분석공간 결과가 존재하는 사건과 엔티티에 한정된다.

요약하면 `eye-lens`는 다중 관점 해석을 위한 강력한 인터페이스지만, 판단의 최종 권위를 가져서는 안 된다.
